# Development Environment

This document describes the development environment I intentionally maintain on my Mac.

My goal is to understand every tool I use, why I use it, and how it supports the software I build.

---

## Operating System

### macOS

Provides the foundation for my entire development environment.

---

## Package Management

### Homebrew

Installs, updates, and manages the development tools I use.

---

## Version Control

### Git

Tracks the history of every project I build.

### GitHub

Stores and shares my Git repositories.

---

## Language Runtimes

### Node.js

Allows me to build and run modern JavaScript applications.

### Python

Supports automation, AI, scripting, and future development projects.

---

## Version Managers

### nvm

Manages Node.js versions across projects.

### pyenv

Manages Python versions across projects.

---

## Development Environment

### Visual Studio Code

My primary code editor.

Theme:
- Dracula Theme
- GitHub Dark Default (installed as an alternative)

Extensions:
- Prettier
- ESLint
- GitHub Copilot
- Material Icon Theme
- Error Lens
- GitLens

### zsh

My primary command-line shell.

---

## Philosophy

Every tool in this environment has a purpose.

Before adding something new, I want to understand:
- What problem does it solve?
- Why am I choosing it?
- How do I maintain it?
- Will I actually use it?

This document will continue to evolve as I build new projects and expand my workflow.
